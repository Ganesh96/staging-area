# Project - Amazon SDE Interview Preparation

## Prep objective
Build interview-readiness for Amazon SDE loops by integrating LP behavioral evidence, coding pattern fluency, and system-design tradeoff articulation.

## Interview tracks
1. DSA/coding track (frequency problems + pattern reinforcement).
2. System design track (component/tradeoff communication).
3. Behavioral LP track (STAR evidence bank mapped to LPs).
4. Resume/project deep-dive track (impact, constraints, decisions, outcomes).

## Weekly cadence
- 3 DSA sessions (pattern-cluster based).
- 2 system design sessions (one whiteboard, one critique).
- 2 behavioral mock sessions (LP-specific story scoring).
- 1 integrated mock (mixed technical + behavioral).

## Daily execution protocol
- Warm-up: 1 medium or targeted weak-topic problem.
- Core: 2 priority tasks from current track.
- Debrief: 10-minute error log + one STAR evidence refinement.
- End-of-day: schedule next day with one non-negotiable high-leverage task.

## Scorecard
- [ ] LP story bank coverage (minimum 2 stories per high-frequency LP).
- [ ] DSA pattern coverage and weak-topic burn-down.
- [ ] System-design articulation score (clarity + tradeoff depth).
- [ ] Mock feedback closure rate.

## Open loops
- Validate role-specific emphasis (SDE I vs SDE II phone screen depth).
- Resolve unreadable PDF corpus for any missed high-value content.

## Blockers
- Large duplicated source set can create planning noise.
- Risk of checklist completion without timed performance validation.
