# Project - DoorDash Interview Preparation

## Current prep objective
Close high-frequency DoorDash problem gaps while rehearsing pattern explanations, tradeoffs, and timed execution.

## Problem-pattern priorities
1. Graph/Matrix traversal (BFS/DFS, multi-source BFS).
2. Heap/scheduling patterns.
3. Tree recursion + serialization patterns.
4. Design patterns (cache/file system/trie).
5. Sliding window / monotonic stack / interval merge.

## Weekly execution cadence
- 3 coding sessions (pattern-focused).
- 2 timed mixed sets (45–60 min each).
- 1 system-design + communication rehearsal session.
- 1 behavioral story rehearsal tied to technical tradeoffs.

## Daily execution cadence
- 1 warm-up (easy/medium).
- 2 priority problems from current weak cluster.
- 10-minute error log + pattern summary update.

## Completion tracking
- [ ] Priority pattern coverage complete.
- [ ] Frequency-list unresolved problems reduced week over week.
- [ ] Timed session accuracy and completion speed tracked.
- [ ] Story bank updated with technical tradeoff narratives.

## Review loop
- End of each week: identify 3 weakest patterns, re-queue them for next week.
- Keep one "last-48-hours" compact revision set for interview window.

## Blockers / open loops
- Overlap/conflict across old/renewed/complete guide schedules.
- Some items appear SQL/non-core coding; may require role-fit filtering.
