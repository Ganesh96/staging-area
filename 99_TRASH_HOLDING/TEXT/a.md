You are to create a comprehensive markdown-formatted document that fully dumps all relevant context information.

The document should include:

- A detailed checklist enumerating all tasks, preparations, and advice associated with the context.
- For each checklist item, indicate completion status:
  - Use a checked box ([x]) for items the user has already completed.
  - If an item is inherently or automatically completed ("on by itself"), represent it with strikethrough formatting.

Structure the output clearly so that the user can easily see what remains to be done and what is already covered.

# Steps

1. Gather all context details regarding the tasks and preparations.
2. Enumerate all necessary actions and advice as individual checklist items.
3. Identify items already completed by the user and represent these with checked boxes.
4. Identify any tasks or conditions inherently fulfilled and mark with strikethrough.
5. Format the entire output as markdown using standard checklist syntax.

# Output Format

- Markdown-formatted text.
- Use `- [ ]` for incomplete tasks.
- Use `- [x]` for completed tasks.
- Use `~~text~~` to show strikethrough for tasks completed by default or "on by itself."

# Examples

- [x] Task completed by user
- [ ] Task yet to be done
- ~~Automatically completed task~~

# Notes

- Assume all information about completed tasks is provided or can be inferred.
- The goal is to provide a clear status overview in markdown.
