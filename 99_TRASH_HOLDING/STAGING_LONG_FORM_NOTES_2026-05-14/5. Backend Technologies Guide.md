# 5. Backend Technologies Guide

#tags: #backend #database #messaging #monitoring #api

This guide serves as an index for the specific backend technologies detailed in your `Guide` and `Doubts` folders. It provides a summary and links for deeper dives.

### API Management

- **[[FastAPI]]**: A modern, fast (high-performance) web framework for building APIs with Python 3.7+ based on standard Python type hints.
    
    - _Key Features_: Automatic interactive documentation (Swagger UI), dependency injection, and asynchronous support.
        
    - _Use Case_: Ideal for building microservices and serving ML models.
        
- **[[Apigee]]**: A comprehensive API management platform (now part of Google Cloud).
    
    - _Key Features_: Provides an API gateway for security (OAuth, API keys), traffic management (rate limiting, throttling), and analytics.
        
    - _Use Case_: Centralizing control and visibility over a large number of APIs in an enterprise setting. Links to **[[4. Architectural Patterns and Principles]]**.
        

### Data Processing & Messaging

- **[[Apache Kafka]]**: A distributed event streaming platform used for high-throughput, fault-tolerant, real-time data feeds.
    
    - _Key Features_: Publish-subscribe model, durable storage, and a scalable, distributed architecture. The backbone of **[[4. Architectural Patterns and Principles|Event-Driven Architectures]]**.
        
- **[[Kafka Streams]]**: A client library for building applications and microservices, where the input and output data are stored in Kafka clusters.
    
    - _Key Features_: Enables real-time processing, joins, and aggregations on Kafka topics without a separate processing cluster.
        
- **[[Apache Spark]]**: A unified analytics engine for large-scale data processing.
    
    - _Key Features_: In-memory cluster computing, support for batch processing (ETL) and real-time streaming, and a powerful ML library (MLlib).
        
- **[[Celery]]**: A distributed task queue for Python.
    
    - _Key Features_: Used to run asynchronous background tasks, such as sending emails or processing data, offloading work from the main application thread.
        

### Databases

- **[[MongoDB]]**: A source-available cross-platform document-oriented database program. Classified as a NoSQL database program, MongoDB uses JSON-like documents with optional schemas.
    
    - _Key Features_: Flexible schema, horizontal scaling (sharding), and a rich query language.
        
- **[[SQL Server - Azure SQL]]**: A deep dive into the differences between on-premises SQL Server and its managed cloud counterpart on Azure.
    
    - _Key Concepts_: RDBMS, Managed Instances, automated backups, and scalability in the cloud.
        

### Monitoring and Observability

- **[[Azure App Insights]]**: An Application Performance Management (APM) service for developers and DevOps professionals.
    
    - _Key Features_: Monitors live applications for performance anomalies, provides distributed tracing for microservices, and tracks user behavior.
        
- **[[Tableau]]**: A leading data visualization tool used for business intelligence.
    
    - _Key Features_: Connects to a wide variety of data sources to create interactive and shareable dashboards.
