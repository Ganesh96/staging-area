# Model - Routine and Time Architecture Stack

## Core model
Design a stable day with four layers: sleep protection, morning anchor, focused work blocks, and evening shutdown.

## Mechanism
- Sleep container preserves energy baseline.
- Morning anchor removes startup friction.
- Time blocks protect attention and separate deep vs shallow work.
- Shutdown + digital sunset closes cognitive loops and improves next-day readiness.

## When to use
- When execution is inconsistent or reactive.
- When decision fatigue and context-switching are high.

## When not to use
- Emergency/crisis days that require dynamic reprioritization.

## Signals / diagnostics
- If mornings become reactive, deep-work protection has failed.
- If sleep/wake drifts repeatedly, system stability is degrading.
