# Model - Reusable Prompt Spec Pattern

## Core model
A reusable prompt system spec is composed of: role definition, operating principles, execution phases, and output constraints.

## Mechanism
Standardizing prompt architecture makes systems portable across domains while preserving consistency.

## When to use
- Designing repeatable AI assistants for learning, coaching, or execution workflows.

## When not to use
- One-off prompts with no reuse intent.

## Signals / diagnostics
- Weak spec: no constraints, no phase logic, no evaluation criteria.
