# Model - Amazon Interview Preparation Stack

## Layers of prep
1. Execution layer: schedule, constraints, scoring loop.
2. Behavioral layer: LP-mapped STAR evidence.
3. Coding layer: pattern clusters + timed reps.
4. System design layer: architecture + tradeoff articulation.
5. Resume/project layer: impact narrative under probing.

## Interaction between layers
- Behavioral evidence quality constrains interviewer trust even with strong coding.
- Coding speed without explanation quality weakens system-design evaluations.
- Execution discipline is the control layer that sustains all tracks.

## Failure modes
- Resource hoarding without scored practice.
- LP stories as anecdotes without measurable impact.
- System design answers that list components but skip tradeoffs.
- No feedback loop between mock sessions and next-day plan.

## Diagnostic questions
- Can you map each mock miss to one layer and one corrective action?
- Which LPs have weak evidence density today?
- Which technical clusters have recurring misses under time pressure?
- Is weekly schedule balanced across all four interview tracks?

## When to use each layer
- Early prep: prioritize execution + coding baseline.
- Mid prep: increase behavioral mapping and design articulation.
- Final prep: integrated mocks and loop-specific calibration.
