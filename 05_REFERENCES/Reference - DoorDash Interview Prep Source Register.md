# Reference - DoorDash Interview Prep Source Register

## Source file routing matrix
| Source file | Role | Status | Routing decision |
|---|---|---|---|
| `Checklist.md` | Primary frequency + completion tracker | active | Canonical execution list |
| `Complete Study Guide.md` | Structured schedule with pattern blocks | active | Canonical scheduling scaffold |
| `ALL Leetcode.md` | Large raw problem inventory | active-support | Use as backlog reservoir, not daily driver |
| `renewed.md` | Newer aggressive schedule variant | active-support | Use for compression/final revision scenarios |
| `old guide.md` | Older schedule variant | superseded | Keep as historical reference only |

## Deduplication notes
- `old guide.md` and `renewed.md` overlap heavily in schedule framing.
- `Complete Study Guide.md` is preferred as the most complete structured plan.
- `Checklist.md` is preferred for completion-state tracking.
