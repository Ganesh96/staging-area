# 6. Frontend and Scripting Guide

#tags: #frontend #scripting #cli #angular #streamlit

This guide covers frontend technologies and essential scripting skills from your `Guide` folder, which are crucial for full-stack development and system administration.

### Frontend Frameworks

- **[[Angular]]**: A platform and framework for building single-page client applications using HTML and TypeScript.
    
    - **Architecture**: Component-based, opinionated framework with built-in dependency injection, routing, and state management solutions (like NgRx).
        
    - **Key Concepts**: Modules, Components, Services, Directives, Two-way Data Binding, RxJS for reactive programming.
        
    - **Use Case**: Ideal for large, enterprise-scale applications where a standardized structure is beneficial. Connects to backends discussed in the **[[5. Backend Technologies Guide]]**.
        

### Data Visualization & Prototyping

- **[[Streamlit]]**: An open-source Python library that makes it easy to create and share beautiful, custom web apps for machine learning and data science.
    
    - **Architecture**: Turns Python scripts into interactive web apps with simple commands.
        
    - **Key Concepts**: Widgets (sliders, buttons), session state, reactive updates, and integration with data libraries like Pandas and visualization libraries like Plotly.
        
    - **Use Case**: Rapidly building data dashboards, ML model demos, and internal tools without needing frontend experience. A perfect companion for the data processing pipelines in the **[[5. Backend Technologies Guide]]**.
        

### System & Command-Line Scripting

- **[[Linux Shell]]**: The essential command-line interface for interacting with Linux/Unix-based systems.
    
    - **Core Utilities**: `grep` (search), `awk` (process text), `sed` (edit text), `curl` (transfer data), `ps` (process status), `dmesg` (kernel messages). These are fundamental for DevOps tasks outlined in the **[[Cloud-Native & DevOps Roadmap]]**.
        
    - **Bash Scripting**: Automating tasks by writing scripts using loops, conditionals, and functions. Essential for CI/CD pipelines and system administration.
        
    - **Performance Monitoring**: Using tools like `top`, `vmstat`, and `iostat` to diagnose system performance issues. This complements application-level monitoring with tools like **[[5. Backend Technologies Guide|Azure App Insights]]**.
