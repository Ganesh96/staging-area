# DSA Learning & Interview Principles

---

## Core DSA Learning Guidance

### 1. Arrays & Strings
- Focus on sliding window and two-pointer patterns.
- Use prefix sums and hashmaps for frequency problems.
- Understand sorting (quick sort, merge sort) and when to apply them.
- Practice string operations: reversing, substrings, pattern matching (KMP).

---

### 2. Linked Lists
- Break problems into primitives: reverse, merge, detect cycle.
- Visualize pointer movement to avoid confusion.
- Handle edge cases (empty list, single node).
- Use Floyd’s Tortoise and Hare for cycle detection.

---

### 3. Stacks & Queues
- Memorize core operations (push, pop, peek).
- Practice monotonic stack/queue problems.
- Understand expression evaluation and infix → postfix conversion.

---

### 4. Trees
- Master recursion for DFS traversals.
- Use BFS for level-order problems.
- Start with binary trees → BSTs → N-ary trees.
- Apply helper functions for LCA, diameter, height problems.

---

### 5. Graphs
- Master DFS, BFS, Dijkstra’s algorithm.
- Study topological sort for DAGs.
- Practice connected components and cycle detection.

---

### 6. Dynamic Programming
- Identify overlapping subproblems and optimal substructure.
- Start with memoization → then tabulation.
- Learn classics: Fibonacci, Knapsack, LCS.
- Clearly define base cases.

---

### 7. Recursion & Backtracking
- Visualize recursion trees.
- Use backtracking for constraint problems.
- Practice N-Queens, Sudoku, permutations, combinations.

---

### 8. Sorting & Searching
- Practice binary search with custom conditions.
- Understand algorithm complexity trade-offs.
- Solve merge-intervals and kth-element problems.

---

### 9. Greedy Algorithms
- Identify local-optimal-choice patterns.
- Practice interval scheduling and job sequencing problems.

---

### 10. Bit Manipulation
- Master XOR, AND, OR patterns.
- Solve single-number, subsets, and bitmask problems.

---

## Interview Strategy

### Pre-Interview
- Balance practice: weak areas + speed.
- Simulate interview conditions with timers.
- Practice across multiple platforms (LeetCode, Codeforces, HackerRank).

---

### During Interview
- Clarify the problem before coding.
- Start with brute force, then optimize.
- Explain reasoning and edge cases.
- Dry-run code with examples.
- Write readable, clean code.

---

### Behavioral Signals
- Think out loud.
- Communicate assumptions.
- Use interviewer hints productively.

---

### Post-Solution
- Explain time and space complexity.
- Discuss trade-offs and alternative approaches.
